Class Spell

Spell class, which is different from core/SpellData. Spells are renderable, moveable entities that directly appears in the game screen, and is responsible for dealing damage / heals from its source to its target.

Spells must have a source, so it must be casted from a mob. Spells will be automatically added into its source's scene.

use SpellFlags.targetingEverything to make the spell to hit with everything.

scrollFactorX

scrollFactorX: number

Inherited from dPhysSprite.scrollFactorX

Overrides dSprite.scrollFactorX

  • Defined in tsDefinitions/phaser.d.ts:56991

The horizontal scroll factor of this Game Object.

The scroll factor controls the influence of the movement of a Camera upon this Game Object.

When a camera scrolls it will change the location at which this Game Object is rendered on-screen. It does not change the Game Objects actual position values.

A value of 1 means it will move exactly in sync with a camera. A value of 0 means it will not move at all, even if the camera moves. Other values control the degree to which the camera movement is mapped to this Game Object.

Please be aware that scroll factor values other than 1 are not taken in to consideration when calculating physics collisions. Bodies always collide based on their world position, but changing the scroll factor is a visual adjustment to where the textures are rendered, which can offset them from physics bodies if not accounted for in your code.

    Sets the scroll factor of this Game Object.

    The scroll factor controls the influence of the movement of a Camera upon this Game Object.

    When a camera scrolls it will change the location at which this Game Object is rendered on-screen. It does not change the Game Objects actual position values.

    A value of 1 means it will move exactly in sync with a camera. A value of 0 means it will not move at all, even if the camera moves. Other values control the degree to which the camera movement is mapped to this Game Object.

    Please be aware that scroll factor values other than 1 are not taken in to consideration when calculating physics collisions. Bodies always collide based on their world position, but changing the scroll factor is a visual adjustment to where the textures are rendered, which can offset them from physics bodies if not accounted for in your code.
